# AGENTS.md

## 行動原則

- 3ステップ以上のタスクは、実装前に計画を提示して合意を得てから着手する
- 動作を証明できる（テスト合格・コマンド出力・実行ログ等の証拠提示）までタスクを完了としない
- 非自明な設計を提案するときは、理想案を起点に立ち位置を明確化して提示する（詳細は「設計案の提示規約」）

## コミュニケーション

- ユーザーの気分を害さないために評価を和らげないでください。
- 事実と推論を明確に区別してください。仮定や不確かな情報は明示的にフラグを立ててください。
- 判断を提示する場合、推奨案とその根拠と代替案を説明すること
- 要件が不明確な場合は推測せず、構造化された質問ツール（AskUserQuestion、request_user_input 等）が利用可能ならそれを使い、なければ通常の質問で内容を明確にすること
