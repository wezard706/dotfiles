# テスト実装

実装パターンのリファレンスを読み込む: `./reference.md`

## ワークフロー

### Step 1: 情報の収集

以下を確認する。

- **テスト設計書**: 会話の文脈またはファイルから取得
- **テスト対象コード**: 対象のクラス・メソッドを読み込む

### Step 2: テストコードの実装

**実装規約（AAA・Arrange の外出し・命名・モック・振る舞いベースのアサーション等）は `./reference.md` が正本**。それに従って実装する。ワークフロー上とくに守る点:

- アウトラインの各 `it` を1つのテストケースとして実装すること
- 設計書（アウトライン）の `context`/`it` 名はそのまま使うこと（勝手に言い換えない）
