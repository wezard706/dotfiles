# Commit

Git Commitの作成を支援します。以下の手順に従ってください。

1. `git status` でワーキングツリーの状態（ステージ済み・未ステージ・untracked）を確認する
2. 変更が一切無ければ、コミットを作成せずユーザーに「変更が無いためコミットを行いません」と通知して処理を終える。`--allow-empty` による空コミットはユーザーが明示的に依頼した場合（例: 「空コミットを作って」「`--allow-empty` で作って」）のみ作成する
3. 変更内容を把握する。tracked ファイルの差分は `git diff HEAD` で確認し、untracked ファイルの中身は必要に応じて Read で確認する
4. 把握した変更を分析し、複数の論理的に独立した変更が含まれているかを判断する
5. 複数の独立した変更が検出された場合、より小さい複数のコミットに分割することをユーザーに提案し、ユーザーの承認（あるいは別案の指示）を得てから次のステップへ進む
6. グループ化した変更それぞれに対して、ステージング（個別パスを `git add` で指定）とコミットを行う。コミットメッセージは `references/commit-message.md` ルールに従うこと
