# テスト実装

実装パターンのリファレンスを読み込む: `./reference.md`

## ワークフロー

### Step 1: 情報の収集

以下を確認する。

- **テスト設計書**: 会話の文脈またはファイルから取得
- **テスト対象コード**: 対象のクラス・メソッドを読み込む

### Step 2: テストコードの実装

リファレンスを参考に、以下の原則に従って実装する。

- **Arrange-Act-Assert（AAA）** パターンで記述すること
- **モックは最小限**に留めること。実際のオブジェクトを使える場合はそちらを優先する
- テストケース一覧の各行を1つのテストケースとして実装すること
- テストケース名は設計書の名称と対応させること
