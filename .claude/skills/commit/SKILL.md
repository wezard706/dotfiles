# Commit

Git Commitの作成を支援します。以下の手順に従ってください。

1. `git status` でステージされているファイルを確認する
2. `git diff HEAD` を実行してコミットする変更内容を把握する
3. diffを分析して、複数の論理的に独立した変更が含まれているかを判断する
4. 複数の独立した変更が検出された場合、より小さい複数のコミットに分割することをユーザーに提案する
5. グループ化した変更それぞれに対して、ステージングとコミットを行う。コミットメッセージは commit-message ルールに従うこと
