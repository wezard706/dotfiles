# Squash

ブランチ内の全コミットを1つにまとめます。以下の手順に従ってください。

## 1. 事前確認

- `git branch --show-current` で現在のブランチ名を確認する
- `git log origin/main..HEAD --oneline` でまとめ対象のコミット一覧を取得する
- コミットが1件以下の場合は「まとめるコミットがありません」と伝えて中止する

## 2. コミットメッセージの生成

- `git log origin/main..HEAD --reverse` で全コミットの詳細を取得する
- `git diff origin/main..HEAD` で変更内容全体を把握する
- 全コミットの内容を統合して、1つのコミットメッセージを生成する
- コミットメッセージは commit-message ルールに従うこと
- squash後のコミットは「最終的な1つのまとまり」として履歴に残る。「バグを修正した」「レビューコメントに対応した」などの反復作業の痕跡は含めず、最終的な変更内容を正確に表現すること

## 3. ユーザーへの確認

以下の情報を提示してAskUserQuestionで確認を求める:

- まとめるコミット数とコミット一覧
- 生成したコミットメッセージ（subject + body の完全な形）

ユーザーが「実行する」を選んだ場合のみ次のステップに進む。
「キャンセル」または「メッセージを修正する」の場合は適切に対応する。

## 4. Squashの実行

```bash
git reset --soft origin/main
git commit -m "<生成したメッセージ>"
```

実行後、`git log --oneline -3` で結果を確認してユーザーに報告する。
