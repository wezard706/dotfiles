# 用語規律

すべての会話と成果物に適用します。プロダクト・機能・仕組みを説明するときの用語の使い方を定めます。

## 造語の抑制

- 既存の名前があるものに新しい名前を付けないこと。コード・ドキュメント・会話に既出の用語を最優先で使うこと
- 出典のない呼び方を説明の便宜上導入する場合は「（便宜的な呼称）」と明示すること。明示せずに既知の用語のように使い回してはならない
- 成果物（設計書・ドキュメント・スキルなど）に残る名前を新しく導入する場合は、確定前にユーザーに確認すること

## 用語集の参照と育成

- プロジェクトに `docs/ubiquitous.md` があれば作業開始時に読み、プロダクト・機能の説明ではその用語を使うこと
- ユーザーから用語の誤りを指摘された、または正しい用語を教わったときは、その場で自分の理解した定義を述べて確認し、`docs/ubiquitous.md` への追記を提案すること

### 用語集フォーマット

```markdown
# 用語集

| 用語 | 定義 |
|------|------|
| 予約枠 | コーチが受講者に公開する予約可能な時間帯 |
```
