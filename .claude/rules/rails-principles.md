# Ruby on Rails コーディング原則

- リッチドメインモデル（Rich Domain Model）を推奨します
- Fat Model は避けて下さい
- ConcernよりもPOROへの委譲を好みます

# Fat Model の回避方法

Fat Modelの原因は1つのモデルが複数の責務を持っている可能性が高いです。
そのため、モデルの責務を分解して適切なクラスに委譲することが効果的です。
以下の手順でモデルの分解を進めて下さい。

1. 概念モデリングに立ち返る。闇雲にクラスを分解すると、概念モデルと実装が一致せず技術負債につながります。概念モデリングを行い、現在の概念モデルを見直してください。
2. デザインパターンを活用し、モデルの分解ができないかを検討する
  - GoFのデザインパターン
  - Value Object
  - Form Object
  - Query Object
  - View Object
  - Policy Object
  - Decorator
3. 分解の方針が立ったらモデルをPOROとして実装する

注意: サービスオブジェクトは非推奨です。利用する場合にはユーザーに確認してください。
